# Module 4: Linux Log Troubleshooting


## About This Module

This module focuses on Linux log analysis and troubleshooting techniques commonly 
used by system administrators and cloud engineers. It emphasizes identifying system 
and service issues using logs as the primary source of evidence.


## What Is Covered

- Understanding Linux logging fundamentals
- Identifying log locations and log types
- Viewing and filtering logs using journalctl
- Analyzing authentication and failed login logs
- Analyzing service error logs
- Troubleshooting service startup failures
- Identifying permission-related issues from logs
- Detecting disk and resource issues via logs
- Monitoring logs in real time
- Documenting troubleshooting findings
- Understanding log rotation and cleanup


## Folder Structure

`README.md` – Module overview and learning scope
`practice.md` – Hands-on log analysis commands and exercises
`incident.md` – Simulated log-based incidents and troubleshooting scenarios
`screenshots/` – Evidence and supporting screenshots


## Key Topics

- Linux logging architecture (syslog vs systemd journal)
- Journalctl usage and filtering techniques
- Authentication and SSH log analysis
- Service and system error investigation
- Permission, disk, and resource-related log patterns
- Log-based troubleshooting workflow
- Basic log maintenance and rotation concepts


## Learning Outcome

After completing this module, the learner is able to analyze Linux 
system and service logs, identify root causes of common incidents, 
and perform structured troubleshooting on Linux servers and cloud virtual machines.
